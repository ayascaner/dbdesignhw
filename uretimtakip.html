Üretim Takip Sistemi Veri tabanı Tasarımı

Tanım:

Proje bazlı talaşlı imalat hizmeti veren bir firmanın üretim takip veri tabanı tasarımı gerçekleştirilecektir.

Firmanın müşteriden gelen siparişlere göre yaptığı üretimleri operatör-makine-iş detayında tutabileceği bir veri tabanı tasarım gerçekleştirilecektir. Gereklilikler aşağıda kısa maddeler halinde anlatılmıştır.

Gereklilikler:

Müşteri firma istemiş olduğu ürünler için resmi bir sipariş formu gönderir. Firmanın izlenebilirliği sağlaması için gerekli olan, bu sipariş formun numarasıdır. Kimi durumlarda bu form gelmeden de üretime başlanabilir. Dolayısıyla müşteri sipariş form numarası daha sonradan ilgili kayıtlarla eşlenebilir olmalıdır.

  1. Proje bazlı faaliyet gösterildiği için her bir siparişe bir proje adı verilmesi gerekmektedir. Aynı proje isimleriyle ilerleyen zamanlarda tekrar üretim gerçekleştirilebileceği için aynı proje isimleri kullanılabilir olacaktır.
  2. Sipariş formunda bulunan her satır bir ürün gurubunu veya sadece bir ürünü ifade edebilmektedir. Dolayısıyla her satırda bulunan siparişlerin detaylandırılabilir olmalıdır.
  3. Müşteri firma istemiş olduğu ürünler için teknik resim göndermektedir, bu teknik resimlerin kayıt edilmesi gerekmektedir. Kimi durumlarda müşteri teknik resim yerine katı model dosyası göndererek talepte bulunmaktadır. Firma herhangi bir aşamada teknik resimlerin kayıtlarını oluşturabilir olacaktır.
  4. Firma üretimi gerçekleştirilecek olan her bir ürün kendisi teknik resim hazırlamaktadır. Dolayısıyla firmanın hazırlamış olduğu teknik resimlerin ayrıca kaydı tutuluyor olacaktır.
  5. Firma sipariş formunda bulunan her bir satırı detaylandırırken kayıt edilmiş olan firma ve müşteri teknik resimleriyle eşleştirecektir.
  6. Sipariş formunda detaylandırılan ürünler kayıt edilirken üretim sırasında kaç operasyondan geçeceği tanımlanacaktır. Bu miktar daha sonra değiştirilebilir olacaktır.
  7. İlgili parça için tanımlanmış operasyon sayısı kadar iş emri hazırlanabilir olmalıdır.
  8. Oluşturulan bütün iş emirleri için ayar başlama zamanı, ayar bitiş zamanı(işleme başlangıcı), çevrim zamanı, iş tamamlanma zamanı tutuluyor olacaktır.
  9. Her bir ürünün ne zaman ve kim tarafından yapıldığı ayrıca kayıt altına alınacaktır.

DATABASE VE TABLOLARI OLUŞTURMA

-- -----------------------------------------------------
-- uretim_takip database olusturulmasi
-- -----------------------------------------------------
DROP DATABASE uretim_takip;
CREATE DATABASE uretim_takip;

-- -----------------------------------------------------
-- sonraki sql querylerde uretim_takip database'yi kullan
-- -----------------------------------------------------
USE uretim_takip;

 


-- -----------------------------------------------------
-- firma tablosunun olusturulması
-- -----------------------------------------------------

CREATE TABLE IF NOT EXISTS firma (
firma_id INT NOT NULL AUTO_INCREMENT,
firma_adi TEXT NOT NULL,
firma_adresi TEXT NOT NULL,
firma_vergi_numarasi INT(10) NOT NULL,
numaralandirma INT NOT NULL,
kayit_tarihi DATE NOT NULL,
PRIMARY KEY (firma_id));

 

-- -----------------------------------------------------
-- tedarikci_firma tablosunun olusturulması
-- -----------------------------------------------------


CREATE TABLE IF NOT EXISTS tedarikci_firma (
fk_tedarikci_firma_id INT NOT NULL UNIQUE,
tedarikci_puani INT(3) NULL,
PRIMARY KEY (fk_tedarikci_firma_id),
CONSTRAINT fk_firma_id
FOREIGN KEY (fk_tedarikci_firma_id)
REFERENCES firma (firma_id));

 


-- -----------------------------------------------------
-- musteri_firma tablosunun olusturulması
-- -----------------------------------------------------

CREATE TABLE IF NOT EXISTS musteri_firma (
fk_musteri_firma_id INT NOT NULL UNIQUE,
alinan_puan INT(3) NULL,
PRIMARY KEY (fk_musteri_firma_id),
CONSTRAINT musteri_id
FOREIGN KEY (fk_musteri_firma_id)
REFERENCES firma (firma_id));

 

 

-- -----------------------------------------------------
-- gorev tablosunun olusturulmasi
-- -----------------------------------------------------

CREATE TABLE IF NOT EXISTS gorev (
gorev_id INT NOT NULL AUTO_INCREMENT,
gorev_adi TEXT NOT NULL,
PRIMARY KEY (gorev_id));

 

-- -----------------------------------------------------
-- personel tablosunun olusturulması
-- -----------------------------------------------------


CREATE TABLE IF NOT EXISTS personel (
tc_kimlik_numarasi BIGINT NOT NULL,
personel_adi TEXT NOT NULL,
personel_soyadi TEXT NOT NULL,
baslangic_tarihi DATE NOT NULL,
calisma_durumu TINYINT NOT NULL,
fk_gorev_id INT NOT NULL,
PRIMARY KEY (tc_kimlik_numarasi),
CONSTRAINT gorev_id
FOREIGN KEY (fk_gorev_id)
REFERENCES gorev (gorev_id));


-- -----------------------------------------------------
-- proje tablosunun olusturulması
-- -----------------------------------------------------

CREATE TABLE IF NOT EXISTS proje (
proje_id INT NOT NULL AUTO_INCREMENT,
proje_adi TEXT NOT NULL,
fk_musteri_firma_id INT NOT NULL,
PRIMARY KEY (proje_id),
